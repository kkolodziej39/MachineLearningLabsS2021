Business Understanding¶

Our dataset on video game sales from kaggle (https://www.kaggle.com/rush4ratio/video-game-sales-with-ratings) contains data about its critic and user scores, the platform it is being released on, the developer and publisher of the game, and their sales in different reigons worldwide. For our classification task, we plan on using our dataset to predict game sales (in units, not dollars) into multiple different ranges. Since the range at which sales for video game units vary so largely, we chose to separate our classification groups in a way that allowed us to best balance the amount of video games in each category. See the visualizations below for a better explanation of why we needed to cut our sales categories this way.

We intend on using this data to forecast unit sales of video games based on the attributes listed above. This does not require real-time forecasting because in order to have a proper forecasting model the user will need to collect user and critic scores as well as basic information about the game. Additionally, sales forecasts take place over months, not seconds, reducing the need for immediate results.

class BinaryLogisticRegression:
    def __init__(self, eta, iterations=20, technique = "steepest", C=0.001, regTerm = "None"):
        self.eta = eta
        self.iters = iterations
        self.technique = technique
        self.C = C
        self.regTerm = regTerm
        # internally we will store the weights as self.w_ to keep with sklearn conventions
        
    def __str__(self):
        if(hasattr(self,'w_')):
            return 'Binary Logistic Regression Object with coefficients:\n'+ str(self.w_) # is we have trained the object
        else:
            return 'Untrained Binary Logistic Regression Object'
        
    # convenience, private:
    @staticmethod
    def _add_bias(X):
        return np.hstack((np.ones((X.shape[0],1)),X)) # add bias term
    
    @staticmethod
    def _sigmoid(theta):
        # increase stability, redefine sigmoid operation
        return expit(theta) #1/(1+np.exp(-theta))
    
    # vectorized gradient calculation with regularization using L2 Norm
    def _get_gradient(self,X,y):
        
        if (self.technique == 'stochGrad'):
            idx = int(np.random.rand()*len(y)) # grab random instance
            ydiff = y[idx]-self.predict_proba(X[idx],add_bias=False) # get y difference (now scalar)
            gradient = X[idx] * ydiff[:,np.newaxis] # make ydiff a column vector and multiply through

            gradient = gradient.reshape(self.w_.shape)
            gradient[1:] += -2 * self.w_[1:] * self.C

            return gradient
        
        elif (self.technique == 'steepest'):
            ydiff = y-self.predict_proba(X,add_bias=False).ravel() # get y difference
            gradient = np.mean(X * ydiff[:,np.newaxis], axis=0) # make ydiff a column vector and multiply through
            return gradient.reshape(self.w_.shape)
        
        elif (self.technique == 'newton'): 
            g = self.predict_proba(X,add_bias=False).ravel() # get sigmoid value for all classes
            hessian = X.T @ np.diag(g*(1-g)) @ X - 2 * self.C # calculate the hessian

            ydiff = y-g # get y difference
            gradient = np.sum(X * ydiff[:,np.newaxis], axis=0) # make ydiff a column vector and multiply through
            gradient = gradient.reshape(self.w_.shape)
            gradient[1:] += -2 * self.w_[1:] * self.C

            return pinv(hessian) @ gradient
    
    # public:
    def predict_proba(self,X,add_bias=True):
        # add bias term if requested
        Xb = self._add_bias(X) if add_bias else X
        return self._sigmoid(Xb @ self.w_) # return the probability y=1
    
    def predict(self,X):
        return (self.predict_proba(X)>0.5) #return the actual prediction
    
    
    def fit(self, X, y):
        Xb = self._add_bias(X) # add bias term
        num_samples, num_features = Xb.shape
        
        self.w_ = np.zeros((num_features,1)) # init weight vector to zeros
        
        # for as many as the max iterations
        for _ in range(self.iters):
            gradient = self._get_gradient(Xb,y)
            self.w_ += gradient*self.eta # multiply by learning rate 
            # add because maximizing 
            # regularization terms
            if(self.regTerm == 'L1' or self.regTerm == 'L1L2'):
                self.w_[1:] -= self.C*abs(self.w_[1:]) # update not include bias
            if(self.regTerm == 'L2' or self.regTerm == 'L1L2'):
                self.w_[1:] += -2 * self.w_[1:] * self.C # taken from class code

class LogisticRegression:
    def __init__(self, eta, iterations=20, technique = "steepest", C=0.0, regTerm = "None"):
        self.eta = eta
        self.iters = iterations
        self.C = C
        self.regTerm = regTerm
        self.technique = technique
        # internally we will store the weights as self.w_ to keep with sklearn conventions
    
    def __str__(self):
        if(hasattr(self,'w_')):
            return 'MultiClass Logistic Regression Object with coefficients:\n'+ str(self.w_) # is we have trained the object
        else:
            return 'Untrained MultiClass Logistic Regression Object'
        
    def fit(self,X,y):
        num_samples, num_features = X.shape
        self.unique_ = np.unique(y) # get each unique class value
        num_unique_classes = len(self.unique_)
        self.classifiers_ = [] # will fill this array with binary classifiers
        
        for i,yval in enumerate(self.unique_): # for each unique value
            y_binary = (y==yval) # create a binary problem
            # train the binary classifier for this class
            blr = BinaryLogisticRegression(self.eta, self.iters, self.technique, self.C, self.regTerm)
            blr.fit(X,y_binary)
            # add the trained classifier to the list
            self.classifiers_.append(blr)
            
        # save all the weights into one matrix, separate column for each class
        self.w_ = np.hstack([x.w_ for x in self.classifiers_]).T
        
    def predict_proba(self,X):
        probs = []
        for blr in self.classifiers_:
            probs.append(blr.predict_proba(X)) # get probability for each classifier
        
        return np.hstack(probs) # make into single matrix
    
    def predict(self,X):
        return self.unique_[np.argmax(self.predict_proba(X),axis=1)] # take argmax along row

def plotHyperparametersVsAccuracy(etas, etaDict, iterations, iterDict, C, cDict):
    fig, (ax1, ax2, ax3) = plt.subplots(1, 3, figsize=(15,5))
    fig.suptitle('Mean Accuracies vs. Hyperparameter')
    etaMeans = [np.mean(accs) for accs in etaDict.values()]
    ax1.plot(etas, etaMeans, 'o')
    ax1.plot(etas, etaMeans)
    ax1.set_xlabel('Eta Value')
    iterMeans = [np.mean(accs) for accs in iterDict.values()]
    ax2.plot(iterations, iterMeans, 'o')
    ax2.plot(iterations, iterMeans)
    ax2.set_xlabel('Iterations')
    cMeans = [np.mean(accs) for accs in cDict.values()]
    ax3.plot(C, cMeans, 'o')
    ax3.plot(C, cMeans)
    ax3.set_xlabel('C value')
    plt.show()    

def plotMaxAccuracies(etas, etaDict, iterations, iterDict, C, cDict):
    fig, (ax1, ax2, ax3) = plt.subplots(1, 3, figsize=(15,5))
    fig.suptitle('Max Accuracies Reached vs. Hyperparameter')    
    etaMax = [max(accs) for accs in etaDict.values()]
    ax1.plot(etas, etaMax, 'o')
    ax1.set_xlabel('Eta Value')
    iterMax = [max(accs) for accs in iterDict.values()]
    ax2.plot(iterations, iterMax, 'o')
    ax2.set_xlabel('Iterations')
    cMax = [max(accs) for accs in cDict.values()]
    ax3.plot(C, cMax, 'o')
    ax3.set_xlabel('C value')
    plt.show()

def bruteForceParameterSearch(etas, iterations, C, regTerms, technique):
    etaDict = {i : [] for i in etas}
    iterDict = {i : [] for i in iterations}
    cDict = {i : [] for i in C}

    highestCombo = (0, (0, 0, 0, 0)) # highest accuracy and corresponding hyperparameters
    for eta in etas :
        for it in iterations :
            for c in C :
                for term in regTerms :
                    lr2 = LogisticRegression(eta=eta, iterations=it, technique=technique, C=c, regTerm=term)
                    lr2.fit(trainX,trainY)
                    yhat = lr2.predict(testX)
                    accuracy = accuracy_score(testY,yhat)
                    etaDict[eta].append(accuracy)
                    iterDict[it].append(accuracy)
                    cDict[c].append(accuracy)
                    if accuracy > highestCombo[0]:
                        highestCombo = (accuracy, (eta, it, c, term))
#                     the print statement below was how we found the best hyper parameters to test on our models
#                     print("Eta:", eta, "Iterations:", it, "C:", c, "RegTerm:", term,'Accuracy of: ', accuracy)
                    
    plotHyperparametersVsAccuracy(etas, etaDict, iterations, iterDict, C, cDict)
    plotMaxAccuracies(etas, etaDict, iterations, iterDict, C, cDict)
    
    return highestCombo

Is brute force searching for the best hyperparameters bad?

There is no doubt that using a brute force approach in this partiuclar scenario where we are unaware of the best ways to find model parameters meets the goal of helping find the most accurate models. However, this sort of brute force approach would likely not be efficient on models that require more computation or are of higher complexity and likely would not be a useful way to tune hyperparameters in those sorts of models. This sort of "data snooping" by altering the hyperparameters to meet the current test and training dataset likely is a cause of some overfitting to the current dataset. It is likely that with our current dataset our hyperparameters are accurately tuned, but with a new testing dataset it would need to change because the accuracy could fall.

It makes sense to deploy newtons model because it is far more consistent at reaching higher levels of accuracy than our other models (see mean accuracy graph).
